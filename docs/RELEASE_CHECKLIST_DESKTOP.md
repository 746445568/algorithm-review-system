# 桌面端发布检查清单（Windows-first）

> 适用范围：`v1` 桌面端（Electron + 本地 `ojreviewd` 服务）。

## 1) 版本号确认

- [ ] 确认本次版本号（例如 `v1.x.y`）与发布计划一致。
- [ ] 校验 `apps/desktop-electron/package.json` 中版本号已更新。
- [ ] 校验发布说明（变更摘要、已知问题）与版本号一致。
- [ ] 确认 Git 标签命名与版本号一致（如 `v1.x.y`）。

## 2) 安装包生成

- [ ] 在干净工作区执行打包（避免脏文件混入）。
- [ ] 生成目标安装工件（至少包含 Windows 安装包/可执行分发件）。
- [ ] 记录构建命令、构建机器信息、构建时间。
- [ ] 确认输出目录仅包含本次目标版本工件。

## 3) 基础启动验证

- [ ] 在未安装开发环境依赖的干净 Windows 环境执行安装。
- [ ] 首次启动应用无崩溃、无白屏、主界面可进入。
- [ ] 关闭后再次启动正常（冷启动 + 热启动各至少 1 次）。
- [ ] 卸载后重装仍可正常启动（验证安装器行为）。

## 4) 本地服务连通验证（127.0.0.1）

- [ ] 应用启动后，本地服务进程被正确拉起。
- [ ] 客户端与本地服务通过 `127.0.0.1` 正常通信。
- [ ] 健康检查接口可达（例如 `/api/health` 返回成功状态）。
- [ ] 服务不可用时，前端有明确错误提示且可恢复。

## 5) 发布工件校验（hash / 体积 / 签名状态）

- [ ] 为每个发布工件生成并记录 hash（建议 SHA256）。
- [ ] 校验工件体积在预期区间内（与上一个版本对比无异常突增）。
- [ ] 校验签名状态（已签名/未签名）并记录证书信息或原因。
- [ ] 下载后复算 hash 与发布页一致。

## 发布结论

- [ ] 以上检查全部通过，允许发布。
- [ ] 若有豁免项，已记录责任人、风险与回滚方案。
