# 修复 Claude Code Hookify JSON 解析错误

## 问题描述

Claude Code 的 hookify 插件在 Windows 上报错：
```
Hookify error: Expecting ',' delimiter: line 1 column 1159 (char 1158)
```

## 根本原因

hookify 插件的 Python 脚本在解析 JSON 时，Windows 文件路径（如 `C:\Users\ackun\...`）中的反斜杠未正确转义，导致 JSON 解析失败。

- 相关 Issue: https://github.com/anthropics/claude-code/issues/20015
- 相关 Issue: https://github.com/anthropics/claude-plugins-official/issues/996

## 解决方案

### 方案 1：临时解决方案（推荐）

用 Node.js 替代 Python 编写 hook 脚本：

1. 在项目 `.claude/` 目录创建 `hooks.json` 配置
2. 使用 Node.js 脚本处理 hook 逻辑

示例 `hooks.json`:
```json
{
  "hooks": {
    "PreToolUse": [{
      "matcher": "Bash",
      "hooks": [{
        "type": "command",
        "command": "node .claude/scripts/your-hook.js"
      }]
    }]
  }
}
```

### 方案 2：等待官方修复

这是 Claude Code 官方 bug，关注 GitHub Issue #20015 和 #996，等待官方发布修复版本。

## 不需要在项目中修复

此问题不是项目配置问题，而是 Claude Code 本身的 bug，与项目内的 `.claude/settings.local.json` 配置无关。
